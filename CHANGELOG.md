## 2026-05-19 ‚Äî Day 3 (in progress)

- **bets-history-and-cancel** ‚Äî Three things land together. (1) New `/bets` route (auth-gated, in ProfileMenu under "Your bets") with a 4-stat strip (total, pending, accuracy, net credits) and two tabs: **Bets** (filterable All/Pending/Resolved with row counts; ticker + direction + stake + status badge + net payout + date per row) and **Credits** (append-only ledger with type badge, signed amount, running balance_after, prediction context when `reference_id` resolves). Helpers `fetchUserBetHistory` + `fetchCreditLedger` + `computeBetStats` in `src/lib/bets.ts`; VOIDs excluded from accuracy denominator. (2) **Cancel-before-lock**: new `cancel_bet(prediction_id)` Postgres RPC mirrors `place_bet` (FOR UPDATE lock + post-lock `resolved=false` check kills the race against the 4:15 PM resolution job). Hard delete on `predictions`; the WAGER+REFUND pair in `credit_transactions` is the audit trail. Locked-in chip becomes clickable while window is open ‚Äî opens BetSheet in manage mode (pencil icon, "Cancel before 1 PM" label) with a two-tap confirm Cancel button. (3) **Smart toast date** for place success: "Resolves today / tomorrow / Mon at 4:15 PM ET" computed in ET so a 9 PM PT bet correctly reads "tomorrow". Also: server actions now `console.error` raw PG errors before mapping, and a missing-function error code surfaces as "Cancel isn't enabled on the server yet (migration pending)" so an unapplied migration is immediately diagnosable. See [ADR 0009 ‚Äî Cancellation](docs/adr/0009-bet-placement-atomicity.md#cancellation-added-2026-05-19). Migration: `20260519000004_cancel_bet_rpc.sql`.
- **bet-placement** ‚Äî End-to-end bet flow lands. New Postgres RPC `place_bet(stock_id, direction, credits, prediction_date)` runs the three writes (insert prediction ¬∑ debit balance ¬∑ append ledger row) in one transaction with a `FOR UPDATE` lock on `user_profiles` ‚Äî concurrent double-clicks can't double-debit. `SECURITY DEFINER` lets the RPC write to `credit_transactions` (whose RLS denies user inserts so the ledger is tamper-proof from the client). Server action `placeBet` (in `src/app/actions/bets.ts`) wraps the RPC, re-checks `MarketSchedule.betWindowOpen` server-side, and maps Postgres errors (`23505` unique-violation, custom `insufficient_credits` / `invalid_credits`) to user copy. New `BetSheet` (base-ui Sheet, bottom on mobile / right drawer on desktop) with direction toggle pre-selected from MarketMind's verdict, 4 stake chips (50/100/250/500) + slider in between, payout preview (WIN +N ¬∑ LOSS ‚àíN ¬∑ TIE refund), live bet-window countdown, and `useTransition` submit spinner. `BetCta` consolidates the three-state CTA (locked-in chip / "Place bet" button / "Window opens in 4h" muted label) used by both StockCard and the detail page. `fetchBetsForTradingDay` keyed on `tradingDayLabel` (not server-local date) means the answer matches what the UI shows. Sonner installed for the "Bet placed ¬∑ 100 on UP" success toast. See [ADR 0009](docs/adr/0009-bet-placement-atomicity.md). Migration: `20260519000003_place_bet_rpc.sql`.
- **schedule-bar-state-aware-headline** ‚Äî `MarketScheduleBar` now leads with a tone-colored, phase-specific headline (live / locked / neutral) so users know what to do *right now* instead of decoding three columns. Seven phases drive copy: `pre-market-bet-open`, `market-open-bet-open`, `market-open-bet-locked`, `post-resolution`, `pipeline-running`, `after-pipeline`, `weekend`. Pipeline messaging models a *window*, not a point: added `PIPELINE_TYPICAL_DURATION_MIN = 25` (~18 min p50 + headroom for HF retries) and `nextPipelineCompletion` to `MarketSchedule`. Copy now reads "land by ~8:25 PM ET" with both start + completion shown. Trading-day cutover waits for *completion*, not the 8 PM cron start, so the UI doesn't claim "tomorrow's predictions are live" while the pipeline is still crunching. New `pipeline-running` phase covers the 8:00‚Äì8:25 window with "you're seeing today's resolved data until it finishes". `/about` cadence section updated to match.
- **onboarding-stock-picker** ‚Äî New `/onboarding` route: sectored stock-grid picker. Sectors as tabs (All / Technology / Financial / Healthcare / etc.). Min 3, max 15 selections. Sticky footer bar shows live count + Continue CTA. Server action commits to `user_watchlist` then redirects home. `src/lib/watchlist.ts` centralizes reads + writes. Home page auto-redirects new users (empty watchlist) here.
- **theme-toggle** ‚Äî Installed `next-themes`. `ThemeProvider` wraps the app with `attribute="class"` + system default. `ThemeToggle` cycles light ‚Üí dark ‚Üí system, mounted in the home-page header. `suppressHydrationWarning` on `<html>` prevents the FOUC. Theme color in viewport meta updated to match each scheme.

## 2026-05-18 ‚Äî Day 2

- **summarizer-fixed** ‚Äî Llama-3-8B-Instruct was failing with opaque `status=?` HTTPErrors ‚Äî the model is gated on HF and the free serverless tier doesn't reliably serve it anymore. Switched to: (1) `chat_completion` API (newer, OpenAI-compatible, routes cleanly), (2) `mistralai/Mistral-7B-Instruct-v0.3` as default (well-supported on free `hf-inference` provider), (3) explicit `provider="hf-inference"` to bypass "auto" routing that lands on paid providers with opaque errors, (4) much richer error logging (exception type + body extract) so future failures debug in one log line. Model is configurable via `HUGGINGFACE_SUMMARY_MODEL` env var.
- **article-summaries-v2** ‚Äî Articles now carry three structured fields (tldr / summary / signal_influence) instead of just one. New migration `20260518000003_article_richer_summary.sql` adds `summary` and `signal_influence` columns to `insight_articles`. `LlamaSummarizer` rewritten with a single labeled prompt that returns all three fields, parsed via regex with defensive trimming; if parsing fails for any field, the others still land. Unit tests in `pipeline/tests/test_summarizer.py` cover the parser's tolerance for mixed case, quotes, trailing punctuation, and partial output. `NewsArticle` dataclass extended; orchestrator inserts the new fields.
- **backtest-harness** ‚Äî `pipeline/backtest.py`: replays the technical bucket against `--months` of historical OHLCV (yfinance + curl_cffi), computes accuracy per score band (high bullish, low bullish, etc.), writes a CSV plus prints a stdout summary. No lookahead ‚Äî each prediction uses only data available at end of evaluation day. Sentiment/professional/social backtests deferred (need historical news + filings ‚Äî Week 2+).
- **resolve-predictions-workflow** ‚Äî `.github/workflows/resolve-predictions.yml`: cron at 21:15 UTC Mon‚ÄìFri (~4:15 PM ET, 15-min cushion for Yahoo to publish the close). `workflow_dispatch` with optional `--date` and `--dry-run`. Same `production` environment, only needs Supabase secrets.
- **resolve-predictions-job** ‚Äî `pipeline/resolve_predictions.py`: fetches unresolved predictions for the target date, fetches each ticker's day bar once via yfinance + curl_cffi, evaluates `direction` vs open‚Üíclose direction. WIN = stake √ó 1.8, LOSS = 0, VOID = stake refund (on flat days or price unavailable). Updates `predictions`, appends to `credit_transactions`, bumps `user_profiles.total_predictions` + `correct_predictions`. Idempotent ‚Äî only touches rows where `resolved=false`.
- **social-signals** ‚Äî `fetchers/stocktwits.py` (public API, no auth, bullish/bearish ratio from messages), `fetchers/apewisdom.py` (one cached request loads top ~500 WSB-mentioned tickers, looked up per-stock), `fetchers/reddit.py` (PRAW-backed mention counts across r/wallstreetbets, r/stocks, r/investing; gracefully skipped if creds absent). Orchestrator's `_build_social` coalesces all three into the `SocialSnapshot` consumed by the aggregator ‚Äî fills the previously-null `social_score` bucket.
- **llama-summarizer** ‚Äî `processors/summarizer.py`: `LlamaSummarizer` uses `huggingface_hub.InferenceClient` to call `meta-llama/Meta-Llama-3-8B-Instruct`. Generates a 1-sentence TL;DR (max 140 chars) for the top 3 articles per stock ‚Äî only the ones we surface in the UI, keeping inference cost low. Mutates `NewsArticle.tldr` in place.

## 2026-05-18

- **fetch-insights-workflow** ‚Äî `.github/workflows/fetch-insights.yml`: cron at 00:00 UTC Tue‚ÄìSat (~8 PM ET Mon‚ÄìFri), plus `workflow_dispatch` with `ticker`/`limit`/`dry_run` inputs for manual testing. Uses the `production` GitHub Environment, pulls all secrets from there (Supabase URL/service key + Massive/Finnhub/FRED/HF keys). Step summary links back to `pipeline_runs` in Supabase for post-run inspection.
- **pipeline-day1-runnable** ‚Äî Wired the full insights pipeline end-to-end. New: `fetchers/finnhub.py` (analyst aggregate + earnings calendar), `fetchers/sec_edgar.py` (Form 4 / 8-K detection with on-demand CIK lookup + 10-req/sec polite UA), `fetchers/fred.py` (VIX macro snapshot, one call shared across all stocks), `processors/sentiment.py` (FinBERT via HF Inference API with recency-weighted aggregation + cross-source agreement counter), `processors/aggregator.py` (the 4 bucket scores per IMPLEMENTATION_PLAN formulas; no aggregate verdict per ADR 0003), and `fetch_insights.py` (the orchestrator ‚Äî parallel per-stock fetch, FinBERT scoring, insight upsert, top-3 article insert, per-source audit, `pipeline_runs` lifecycle). CLI supports `--ticker NVDA --dry-run` for one-shot smoke tests. Sources gracefully degrade ‚Äî missing API keys just skip that fetcher.
- **ui-polish-v1** ‚Äî Fixed the circular `--font-sans` CSS variable in `globals.css` that caused the entire app to render in serif default. Default theme is now dark (better fit for a trading app). Login page rebuilt around a glassmorphism card with proper hero, branded MarketMind logo (gradient + line-chart icon), and a `GoogleSignInButton` matching Google's brand guidelines (white surface, multicolor G logo SVG, "Sign in with Google" copy). Home page now has a sticky header with credit pill, user avatar initial, stat grid (credits + streak + predictions today), and footer disclaimer. Metadata updated from "Create Next App" to proper MarketMind tags.
- **actions-node24** ‚Äî Bumped `actions/checkout@v4 ‚Üí @v6` and `supabase/setup-cli@v1 ‚Üí @v2` in both migration workflows. Clears the Node.js 20 deprecation warning ahead of GitHub's 2026-06-02 cutover; both new major versions run on Node 24.
- **seed-as-migration** ‚Äî Converted `supabase/seed.sql` into migration `20260518000002_seed_stocks.sql` (idempotent via `ON CONFLICT DO NOTHING`). Removes the failure-prone raw-psql seed step from the apply-migrations workflow; now the whole chain runs via `supabase db push`. Local dev still picks it up automatically through `supabase db reset`.
- **ci-migrations** ‚Äî Enterprise-grade migration pipeline. Two workflows: `validate-migrations.yml` runs on PR (spins up local Supabase, applies all migrations from scratch). `apply-migrations.yml` is manual-only via `workflow_dispatch`, requires typing literal `migrate` confirmation, runs dry-run before apply. Added `supabase/config.toml` for CLI + ADR 0006 documenting the design. Docs updated in SETUP + RUNBOOK.
- **auth-flow** ‚Äî Supabase auth integration via `@supabase/ssr`. Browser + server clients (`src/lib/supabase/{client,server}.ts`), session-refresh proxy at `src/proxy.ts` (Next.js 16 file convention ‚Äî formerly `middleware.ts`), OAuth callback at `src/app/auth/callback/route.ts`, Google Sign-In login page at `/login` with Suspense-wrapped form, auth-gated home at `/` showing email + credit balance + streak. Sign-out button as a client component.
- **next16-pipeline-pattern** ‚Äî Discovered Next.js 16 renamed `middleware.ts` ‚Üí `proxy.ts` (the function is `proxy()` not `middleware()`). Documented in code comments; no ADR since it's a forced convention.
